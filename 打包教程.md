# PurgeC 打包为小体积 EXE

以下步骤在 Windows 的项目目录中执行。建议使用 **Python 3.10 或更高版本的 64 位版**；不要把虚拟环境、日志或测试文件打进安装包。

## 1. 创建干净的构建环境

```powershell
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt pyinstaller
```

如果 `py` 命令不可用，请先从 [Python 官网](https://www.python.org/downloads/windows/) 安装 Python，并在安装界面勾选 **Add python.exe to PATH**。

## 2. 先运行检查

```powershell
python -m compileall -q main.py cleaner.py scanner.py migrate.py gui
python main.py
```

手动确认界面和迁移功能后，关闭程序再执行打包命令。

## 3. 生成单文件 EXE

```powershell
pyinstaller --noconfirm --clean --onefile --windowed --name PurgeC --collect-all sv_ttk main.py
```

成品是 `dist\PurgeC.exe`。`--windowed` 不显示黑色控制台窗口，`--onefile` 便于分发且不会额外附带 DLL 目录。

## 4. 兼顾体积与启动速度的选择

单文件模式会在每次启动时解压，因此启动稍慢；若更看重启动速度，改用：

```powershell
pyinstaller --noconfirm --clean --onedir --windowed --name PurgeC --collect-all sv_ttk main.py
```

不要使用 `--collect-all` 收集未使用的包，也不要使用 UPX 压缩系统 DLL；前者会明显增大体积，后者容易被杀毒软件误报。当前项目只需 `send2trash` 和 `sv-ttk`，保持依赖精简即可。

## 5. 分发前检查

在另一台 Windows 或干净的测试账户中运行 `dist\PurgeC.exe`。迁移功能只在用户明确勾选、选择其他分区并二次确认后才会操作数据；历史记录会保存到 `%LOCALAPPDATA%\PurgeC\migrate_history.json`，不写入 EXE 所在目录。
